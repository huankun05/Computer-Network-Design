## 3. 三大功能模块

### 模块一：单项协议原理可视化展示（20 分）

从以下 6 个题目中**任选 2 个**实现，每个协议一个独立可视化页面。

#### 公共页面要素（每个协议页面必须包含）

| 要素 | 说明 |
|------|------|
| 网络拓扑图 | 简洁的网络拓扑，展示协议涉及的设备及其连接关系 |
| 控制按钮 | "开始"、"下一步"、"重置" 等，支持用户控制展示进度 |
| 步骤展示 | 清晰展示协议执行的每一步骤 |
| 动画传递 | 报文或数据帧在拓扑中的传递过程（移动动画） |
| 状态展示 | 关键状态表或结果展示（如缓存表、状态机等） |

#### 可选题目详情

**题目 1：DNS 域名解析**
- 客户端、DNS 服务器和目标域名输入框
- 用户输入域名后，展示 DNS 查询过程
- 展示缓存命中或缓存未命中的情况
- 展示最终解析出的 IP 地址
- 可展示 DNS 缓存表的查询和更新结果

**题目 2：TCP 三次握手与四次挥手**
- 客户端和服务器
- "建立连接"和"释放连接"按钮
- SYN、ACK、FIN 等报文在客户端和服务器之间的传递过程
- TCP 连接状态变化（CLOSED → SYN-SENT → ESTABLISHED → FIN-WAIT 等）
- 最终展示连接建立或连接释放的结果

**题目 3：ARP 地址解析**
- 源主机、目标主机和交换机
- 源主机已知目标 IP、未知目标 MAC 的初始状态
- ARP Request 广播过程
- ARP Reply 返回过程
- ARP 缓存表写入 IP 与 MAC 的映射结果

**题目 4：交换机转发与 MAC 地址表学习**
- 多台主机和一台交换机
- 主机发送数据帧的过程
- 交换机学习源 MAC 地址的过程
- 目的 MAC 未知时的泛洪过程
- 目的 MAC 已知后的定向转发过程
- MAC 地址表的动态变化

**题目 5：路由转发**
- 至少两个不同网段、主机和路由器
- 数据包从一个网段发送到另一个网段的过程
- 路由器查询路由表的过程
- 下一跳选择和数据包转发路径
- 路由表内容及转发结果

**题目 6：FTP 文件传输**
- 至少包含客户端、FTP 服务器和待传输文件
- 展示客户端与 FTP 服务器建立控制连接的过程
- 展示用户登录认证过程（用户名和密码验证）
- 展示文件上传或下载过程中数据连接的建立与传输路径
- 展示控制连接与数据连接的区别，以及文件传输结果

### 模块二：综合网络场景模拟（20 分）

#### 网络拓扑（固定）

```
                        Internet
                           |
                      Web 服务器
                    (www.abc.com)
                           |
                        [R] 路由器
                    192.168.1.1/25
                    00-11-22-33-44-aa
                           |
                        [S] 交换机
                       /   |    \     \
                  端口1  端口2  端口3  端口4
                    |      |      |      |
               DNS服务器   H2     H1
          192.168.1.126/25   |   192.168.1.3/25
          00-11-22-33-44-bb  |   00-11-22-33-44-dd
                            |
                      192.168.1.2/25
                      00-11-22-33-44-cc
```

#### 场景

H1（192.168.1.2）在 **t0** 时刻使用浏览器访问 www.abc.com。
**初始条件**：H1 的 ARP 表为空，交换机 S 的交换表为空。t0 到 t1 期间，网络中无与此次 Web 访问无关的通信。

#### 交付要求

1. **通信过程拆解与阶段划分**
   - 分析 t0 → t1 网络中依次发生的通信行为
   - 拆解为若干连续步骤
   - 明确每个步骤的起点、终点、通信协议、通信目的
   - 明确该步骤结束后网络状态的变化

2. **分步骤可视化展示**（每个步骤需展示以下全部内容）

   | 序号 | 展示内容 |
   |------|----------|
   | 1 | 当前步骤编号和步骤名称 |
   | 2 | 当前通信的起点和终点 |
   | 3 | 当前通信使用的协议（ARP、DNS、HTTP、TCP 等） |
   | 4 | 当前通信是广播还是单播 |
   | 5 | 当前帧或报文的源 MAC 地址和目的 MAC 地址 |
   | 6 | 当前数据报的源 IP 地址和目的 IP 地址 |
   | 7 | 当前数据在拓扑图中的传输路径 |
   | 8 | 交换机 S 的处理行为 |
   | 9 | 当前步骤结束后，交换机 S 的交换表变化 |
   | 10 | 当前步骤结束后，H1 的 ARP 表变化 |
   | 11 | 交换表和 ARP 表动态变化展示 |

3. **最终结果展示**：完成所有步骤后，给出从 t0 到 t1 的最终状态和结论

> 页面应支持用户**逐步查看**从 t0 到 t1 的完整过程，能够**控制展示进度**。

## 8. 通信过程拆解参考（模块二）

H1 访问 www.abc.com 从 t0 到 t1 可能涉及的通信步骤（供参考，需自行分析验证）：

1. H1 构造 HTTP 请求 → 发现需要 DNS 解析
2. H1 检查本地 DNS 缓存 → 未命中
3. H1 构造 ARP 请求（目标：DNS 服务器 192.168.1.126）→ 广播
4. DNS 服务器回复 ARP Reply → H1 更新 ARP 缓存
5. H1 向 DNS 服务器发送 DNS 查询请求
6. DNS 服务器检查缓存 → 未命中 → 向根域名服务器/递归查询
7. DNS 服务器回复 DNS 响应（www.abc.com → IP）
8. H1 需要与路由器通信 → ARP 请求网关 MAC
9. 路由器回复 ARP Reply
10. H1 构造 HTTP 请求 → 发送到路由器
11. 路由器转发 → 交换机学习/转发
12. ...（后续经 Internet 到达 Web 服务器）

> 实际步骤拆解应基于协议原理自行分析，确保每个步骤的源/目的 MAC、IP、协议类型等信息准确。
