# DNS 协议（域名系统）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | DNS（Domain Name System） |
| 所属层 | 应用层 |
| 传输层协议 | UDP（通常）/ TCP（区域传送、大响应） |
| 默认端口 | 53 |
| RFC 文档 | RFC 1034, RFC 1035 |

## 协议简介
DNS 是互联网的核心基础设施之一，负责将人类可读的域名（如 www.example.com）解析为计算机可识别的 IP 地址（如 93.184.216.34）。DNS 采用分布式数据库架构，全球由众多 DNS 服务器协作完成解析。

## 域名层次结构
```
. (根域)
├── .com (顶级域)
│   ├── example.com (二级域)
│   │   └── www.example.com (主机名)
├── .org
├── .net
├── .cn (国家/地区顶级域)
│   └── edu.cn
│       └── cup.edu.cn
│           └── www.cup.edu.cn
```

## DNS 服务器类型
| 类型 | 说明 |
|------|------|
| 根域名服务器 | 全球共 13 组（A-M），负责引导查询到顶级域名服务器 |
| 顶级域名服务器 | 管理 .com、.org、.net 等顶级域 |
| 权威域名服务器 | 管理具体域名的解析记录 |
| 本地 DNS 服务器 | 由 ISP 提供，负责递归查询 |
| 递归解析器 | 接受客户端查询，代替客户端完成全部解析 |

## 查询类型
### 递归查询
客户端向本地 DNS 服务器发出查询请求，本地 DNS 服务器负责完成全部解析工作并返回最终结果。客户端只需发出一次请求。

### 迭代查询
本地 DNS 服务器依次向根域名服务器、顶级域名服务器、权威域名服务器发起查询，每次返回下一级服务器的地址，直到获得最终的 IP 地址。

## DNS 记录类型
| 类型 | 说明 | 示例 |
|------|------|------|
| A | 域名 → IPv4 地址 | example.com → 93.184.216.34 |
| AAAA | 域名 → IPv6 地址 | example.com → 2606:2800:220:1::248 |
| CNAME | 别名记录 | www.example.com → example.com |
| MX | 邮件交换记录 | example.com → mail.example.com (优先级 10) |
| NS | 域名服务器记录 | example.com → ns1.example.com |
| TXT | 文本记录 | 用于 SPF、DKIM 等验证 |
| PTR | 反向解析记录 | IP → 域名 |
| SOA | 起始授权机构记录 | 包含区域的管理信息 |

## DNS 报文格式
DNS 报文分为头部和正文两部分：
- 头部（12 字节）：标识符、标志位、问题数、答案数、权威数、附加数
- 问题部分：查询域名、查询类型、查询类
- 答案部分：域名、类型、类、TTL、数据长度、数据

## DNS 缓存
DNS 缓存分布在多个层级：
1. 浏览器缓存：浏览器保存最近的 DNS 解析结果
2. 操作系统缓存：OS 的 DNS 解析器缓存
3. 路由器缓存：家庭路由器的 DNS 缓存
4. ISP DNS 缓存：本地 DNS 服务器的缓存

TTL（Time To Live）控制缓存的有效时间，单位为秒。

## 与其他协议的关系
- DNS 使用 UDP 53 端口进行常规查询，使用 TCP 53 端口进行区域传送和大于 512 字节的响应
- HTTP 等应用层协议依赖 DNS 进行域名解析
- DNS 解析结果被缓存在多个层级以提高效率
