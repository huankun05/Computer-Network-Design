# FTP 协议（文件传输协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | FTP（File Transfer Protocol） |
| 所属层 | 应用层 |
| 传输层协议 | TCP |
| 控制连接端口 | 21 |
| 数据连接端口 | 20 |
| RFC 文档 | RFC 959 |

## 协议简介
FTP 是最早的互联网协议之一，用于在客户端和服务器之间进行文件传输。FTP 的最大特点是使用两个独立的 TCP 连接：控制连接（端口 21）和数据连接（端口 20）。

## 双连接机制
| 连接类型 | 端口 | 用途 | 生命周期 |
|----------|------|------|---------|
| 控制连接 | 21 | 传输命令和响应（如登录、列目录） | 整个会话期间保持 |
| 数据连接 | 20 | 传输文件数据 | 每次传输时建立 |

## 工作模式
### 主动模式（PORT）
1. 客户端随机开放一个端口 N，连接服务器 21 端口
2. 客户端发送 PORT 命令，告诉服务器自己的数据端口 N
3. 服务器从 20 端口主动连接客户端的 N 端口
4. 数据传输完成后关闭数据连接

### 被动模式（PASV）
1. 客户端随机开放一个端口 M，连接服务器 21 端口
2. 客户端发送 PASV 命令
3. 服务器随机开放一个端口 P（>1024），告诉客户端
4. 客户端从自己的 M 端口连接服务器的 P 端口
5. 数据传输完成后关闭数据连接

## 登录认证
FTP 支持两种登录方式：
- 匿名登录：用户名为 anonymous，密码为空或任意邮箱地址
- 正常登录：使用注册的用户名和密码

## 常用命令
| 命令 | 说明 |
|------|------|
| USER | 指定用户名 |
| PASS | 指定密码 |
| LIST | 列出目录内容 |
| RETR | 下载文件 |
| STOR | 上传文件 |
| CWD | 切换目录 |
| PWD | 显示当前目录 |
| DELE | 删除文件 |
| QUIT | 退出会话 |

## 与其他协议的关系
- FTP 使用 TCP 提供可靠传输
- FTP 的控制连接和数据连接是独立的，这是与其他协议的重要区别
- SFTP（SSH File Transfer Protocol）是 FTP 的安全替代品，使用 SSH 加密
