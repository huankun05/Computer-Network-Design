# IMAP 协议（互联网消息访问协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | IMAP（Internet Message Access Protocol） |
| 所属层 | 应用层 |
| 传输层协议 | TCP |
| 默认端口 | 143（非加密）/ 993（IMAPS） |
| RFC 文档 | RFC 3501（IMAP4rev1） |

## 协议简介
IMAP 允许用户在**服务器端**管理和操作邮件，无需下载到本地。与 POP3 不同，IMAP 支持多设备同步、服务器端搜索、文件夹管理等高级功能。

## 核心优势：邮件保留在服务器
```
手机 ──IMAP──┐
             ├──→ 邮件服务器 ← 所有邮件集中存储
电脑 ──IMAP──┘     (已读/未读、文件夹同步)
```

无论在哪台设备上读邮件、标记已读、移动文件夹，其他设备都会同步看到。

## 常用 IMAP 命令
| 命令 | 说明 |
|------|------|
| LOGIN | 登录认证 |
| SELECT | 选择邮箱文件夹 |
| FETCH | 获取邮件内容/属性 |
| SEARCH | 在服务器端搜索邮件 |
| STORE | 修改邮件标志（已读/星标等） |
| CREATE | 创建文件夹 |
| DELETE | 删除文件夹 |
| LIST | 列出所有文件夹 |
| APPEND | 上传邮件到服务器 |
| LOGOUT | 退出 |

## IMAP 协议特点

### 服务器端搜索
```
客户端 → SEARCH SUBJECT "课程设计"
服务器 → 返回匹配的邮件 ID 列表
```

不用下载全部邮件，直接在服务器端完成搜索。

### 部分下载
```
客户端 → FETCH 1 (BODY.PEEK[HEADER])     ← 只下载邮件头
客户端 → FETCH 1 (BODY.PEEK[TEXT])        ← 只下载正文
客户端 → FETCH 1 (BODY.PEEK[2])           ← 只下载第 2 个附件
```

节省带宽，不需要下载整个邮件。

### 邮件标志（Flags）
| 标志 | 说明 |
|------|------|
| \Seen | 已读 |
| \Answered | 已回复 |
| \Flagged | 星标 |
| \Deleted | 已删除（标记） |
| \Draft | 草稿 |

## IMAP vs POP3 对比
| 特性 | IMAP | POP3 |
|------|------|------|
| 邮件存储 | 服务器端 | 下载到本地 |
| 多设备访问 | ✓ 同步 | ✗ 不友好 |
| 服务器搜索 | ✓ | ✗ |
| 部分下载 | ✓ | ✗ |
| 文件夹管理 | ✓ | ✗ |
| 服务器存储 | 大 | 小 |
| 典型端口 | 143/993 | 110/995 |

## 适用场景
- 多设备（手机 + 电脑 + 平板）管理邮件
- 需要服务器端搜索和分类
- 企业/团队协作场景
