# UDP 协议（用户数据报协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | UDP（User Datagram Protocol） |
| 所属层 | 传输层 |
| 特点 | 无连接、不可靠、面向报文、无拥塞控制 |
| 首部长度 | 固定 8 字节 |
| RFC 文档 | RFC 768 |

## UDP 首部格式
```
 0      7 8     15 16    23 24    31
+--------+--------+--------+--------+
|     源端口 (16)   |   目的端口 (16)   |
+--------+--------+--------+--------+
|     长度 (16)     |    校验和 (16)    |
+--------+--------+--------+--------+
|              数据                  |
+-----------------------------------+

总长度: 仅 8 字节！
```

对比 TCP 首部最小 20 字节，UDP 首部仅 8 字节，开销极小。

## UDP 的特点

### 无连接
- 发送数据前不需要建立连接
- 直接发送数据报
- 接收方无需确认

### 不可靠
- 不保证数据到达
- 不保证顺序
- 不保证不重复
- 不进行拥塞控制

### 面向报文
- 保留应用层报文边界
- 每次发送一个完整数据报
- TCP 是面向字节流的

### 无拥塞控制
- 可以按任意速率发送
- 适合实时应用（不能因拥塞控制而延迟）

## TCP vs UDP 全面对比
| 特性 | TCP | UDP |
|------|-----|-----|
| 连接方式 | 面向连接 | 无连接 |
| 可靠性 | 可靠（确认+重传） | 不可靠 |
| 顺序 | 有序 | 无序 |
| 首部开销 | 20-60 字节 | 8 字节 |
| 流量控制 | 滑动窗口 | 无 |
| 拥塞控制 | 慢启动、拥塞避免等 | 无 |
| 数据传输 | 字节流 | 报文 |
| 适用场景 | 网页、文件、邮件 | 视频、语音、DNS |

## UDP 的典型应用
| 协议/应用 | 为什么用 UDP |
|-----------|-------------|
| DNS | 查询报文小，TCP 握手开销太大 |
| DHCP | 客户端没有 IP 地址时无法建立 TCP 连接 |
| RIP | 路由表更新，需要广播/组播 |
| 视频直播 | 丢失几帧可以接受，延迟更重要 |
| VoIP（网络电话） | 实时性 > 可靠性 |
| 在线游戏 | 低延迟，少量丢包可接受 |
| TFTP | 简单文件传输，UDP 实现简单 |

## UDP 伪首部（用于校验和计算）
UDP 校验和计算需要包含一个"伪首部"，其中包含 IP 层的源/目的地址：
```
+-----------------------------------+
|          源 IP 地址                |
+-----------------------------------+
|         目的 IP 地址               |
+-----------------------------------+
| 全零 | 协议(17) |  UDP 长度        |
+-----------------------------------+
```

这样计算出的校验和可以同时检测 IP 地址错误。
