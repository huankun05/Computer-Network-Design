# ARP 协议（地址解析协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | ARP（Address Resolution Protocol） |
| 所属层 | 网络层（跨越数据链路层） |
| 传输方式 | 广播请求 + 单播响应 |
| 报文类型 | 请求（Request）/ 响应（Reply） |
| RFC 文档 | RFC 826 |

## 协议简介
ARP 用于将网络层的 IP 地址解析为数据链路层的 MAC 地址。当主机需要向同一局域网内的另一台主机发送数据时，如果只知道目标 IP 地址而不知道 MAC 地址，就需要使用 ARP 协议进行解析。

## ARP 工作过程
```
源主机 A (192.168.1.2, MAC: AA)  想发送数据给  目标主机 B (192.168.1.3, MAC: BB)
但 A 的 ARP 表中没有 B 的 MAC 地址

步骤 1: A 构造 ARP 请求广播帧
  - 源 MAC: AA
  - 目的 MAC: ff-ff-ff-ff-ff-ff（广播）
  - 源 IP: 192.168.1.2
  - 目的 IP: 192.168.1.3
  - 消息: "谁的 IP 是 192.168.1.3？请告诉 192.168.1.2"

步骤 2: 局域网内所有主机收到广播帧
  - 主机 C (192.168.1.4): IP 不匹配，丢弃
  - 主机 B (192.168.1.3): IP 匹配，准备响应

步骤 3: B 构造 ARP 响应单播帧
  - 源 MAC: BB
  - 目的 MAC: AA
  - 源 IP: 192.168.1.3
  - 目的 IP: 192.168.1.2
  - 消息: "192.168.1.3 的 MAC 地址是 BB"

步骤 4: A 收到响应，将 IP-MAC 映射写入 ARP 缓存表
```

## ARP 缓存表
ARP 缓存表存储 IP 地址到 MAC 地址的映射关系：
| IP 地址 | MAC 地址 | 类型 | 过期时间 |
|---------|---------|------|---------|
| 192.168.1.3 | 00-11-22-33-44-dd | 动态 | 20 分钟 |
| 192.168.1.1 | 00-11-22-33-44-aa | 静态 | 永久 |

- 动态条目：通过 ARP 协议自动学习，有 TTL 超时机制
- 静态条目：手动配置，不会过期

## ARP 报文格式
| 字段 | 长度 | 说明 |
|------|------|------|
| 硬件类型 | 2 字节 | 网络类型（以太网=1） |
| 协议类型 | 2 字节 | 协议类型（IPv4=0x0800） |
| 硬件地址长度 | 1 字节 | MAC 地址长度（6） |
| 协议地址长度 | 1 字节 | IP 地址长度（4） |
| 操作码 | 2 字节 | 请求=1，响应=2 |
| 发送方 MAC | 6 字节 | 发送方的 MAC 地址 |
| 发送方 IP | 4 字节 | 发送方的 IP 地址 |
| 目标 MAC | 6 字节 | 目标 MAC 地址（请求时为 0） |
| 目标 IP | 4 字节 | 目标 IP 地址 |

## ARP 的变种
| 协议 | 说明 |
|------|------|
| RARP | 反向 ARP，MAC → IP（已被 BOOTP/DHCP 取代） |
| 代理 ARP | 路由器代替目标主机回复 ARP 请求 |
| 免费 ARP | 主机发送 ARP 请求查询自己的 IP，用于检测 IP 冲突 |
| VARP | VRRP 使用的 ARP 变种 |

## ARP 与其他协议的关系
- ARP 在 IP 之下工作，服务于 IP 到 MAC 的映射
- 在综合场景中，H1 首先需要通过 ARP 获取 DNS 服务器和网关的 MAC 地址
- ARP 缓存表的初始状态为空，需要通过 ARP 请求动态学习
