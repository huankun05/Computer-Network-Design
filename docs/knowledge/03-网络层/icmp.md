# ICMP 协议（互联网控制消息协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | ICMP（Internet Control Message Protocol） |
| 所属层 | 网络层 |
| 封装于 | IP 数据报（协议号 = 1） |
| RFC 文档 | RFC 792 |

## 协议简介
ICMP 是 IP 协议的辅助协议，用于在 IP 网络中发送**错误报告**和**网络诊断信息**。ICMP 报文封装在 IP 数据报中传输，但 ICMP 本身不属于传输层协议。

## ICMP 报文分类

### 差错报告报文
| 类型 | 代码 | 说明 |
|------|------|------|
| 3 | 0-15 | 目的不可达（网络不可达/主机不可达/端口不可达等） |
| 4 | 0 | 源端抑制（通知降低发送速率，已废弃） |
| 5 | 0-3 | 重定向（更好的路由可用） |
| 11 | 0 | TTL 超时（TTL 减到 0，报文被丢弃） |
| 12 | 0-2 | 参数问题（IP 首部错误） |

### 查询报文
| 类型 | 代码 | 说明 |
|------|------|------|
| 0 | 0 | Echo Reply（回显应答） |
| 8 | 0 | Echo Request（回显请求） |
| 13 | 0 | 时间戳请求 |
| 14 | 0 | 时间戳应答 |

## ping 命令原理
```
源主机                          目标主机
  |                               |
  | ICMP Echo Request (类型 8) →  |
  |                               |
  | ← ICMP Echo Reply (类型 0)    |
  |                               |
  
计算往返时间 RTT = 回显应答时间 - 回显请求时间
```

ping 命令不依赖传输层协议，直接使用 ICMP 测试网络连通性。

## traceroute 原理
```
源主机 ──→ 路由器1 ──→ 路由器2 ──→ 路由器3 ──→ 目标

TTL=1: 路由器1 返回 ICMP 超时 (类型11)
TTL=2: 路由器2 返回 ICMP 超时 (类型11)
TTL=3: 路由器3 返回 ICMP 超时 (类型11)
TTL=N: 目标主机返回 ICMP Echo Reply (类型0) 或端口不可达 (类型3)
```

通过逐跳增加 TTL 值，traceroute 可以发现路径上的每一跳路由器。

## 常见 ICMP 丢包原因
| ICMP 错误 | 可能原因 |
|-----------|---------|
| 目的网络不可达 | 路由表中无匹配路由 |
| 目的主机不可达 | 主机不在线或 ARP 解析失败 |
| 目的端口不可达 | 目标端口无服务监听 |
| TTL 超时 | TTL 减到 0（可能路由环路） |
| 需要分片但 DF 置位 | 路径 MTU 小于报文大小 |

## 不允许发送 ICMP 差错报告的情况
- ICMP 差错报告报文本身出错
- 分片后的 IP 数据报（非第一片）
- 组播地址的数据报
- 特殊地址的数据报（127.0.0.0/8, 0.0.0.0/8）
