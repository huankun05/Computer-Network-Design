# RARP 协议（反向地址解析协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | RARP（Reverse Address Resolution Protocol） |
| 所属层 | 网络层 |
| 传输方式 | 广播请求 + 单播响应 |
| 报文类型 | 请求（3）/ 响应（4） |
| RFC 文档 | RFC 903 |

## 协议简介
RARP 的功能与 ARP **正好相反**：ARP 是 IP → MAC，RARP 是 MAC → IP。当一台主机只知道自己的 MAC 地址但不知道 IP 地址时（如无盘工作站启动时），使用 RARP 向 RARP 服务器获取 IP 地址。

## RARP 工作过程
```
无盘工作站                         RARP 服务器
  (知道 MAC, 不知道 IP)             (维护 MAC↔IP 映射表)

  1. RARP 请求（广播）
     源 MAC: AA:BB:CC:DD:EE:FF
     目的 MAC: ff-ff-ff-ff-ff-ff
     "我的 MAC 是 AA:BB:CC:DD:EE:FF，我的 IP 是什么？"
──────────────────────────────────────────→

  2. RARP 响应（单播）
     目的 MAC: AA:BB:CC:DD:EE:FF
     查找表中的对应记录
     "你的 IP 是 192.168.1.100"
←──────────────────────────────────────────
```

## RARP vs ARP
| 特性 | ARP | RARP |
|------|-----|------|
| 功能 | IP → MAC | MAC → IP |
| 请求方知道 | IP 地址 | MAC 地址 |
| 请求方不知道 | MAC 地址 | IP 地址 |
| 操作码 | 1（请求）/ 2（响应） | 3（请求）/ 4（响应） |
| 需要服务器 | 不需要（目标主机回复） | 需要 RARP 服务器 |
| 当前状态 | 广泛使用 | 已被 DHCP 取代 |

## RARP 的局限性
1. **需要专门的 RARP 服务器**：每个网段都要部署
2. **只能获取 IP 地址**：不支持子网掩码、网关、DNS 等其他配置
3. **广播通信**：不能跨路由器，每个网段都需要独立服务器
4. **低层协议**：在数据链路层广播，被路由器隔离

## 被 DHCP 取代
由于上述局限性，RARP 已被 **BOOTP** 取代，BOOTP 又被 **DHCP** 取代：

```
RARP → BOOTP → DHCP
(仅IP)  (IP+配置) (IP+租约+丰富选项)
```

现代 DHCP 提供了 RARP 的全部功能并远远超越了它：
- 自动分配 IP + 子网掩码 + 网关 + DNS
- IP 地址租约管理
- 跨网段中继
- 丰富的配置选项

## RARP 的历史意义
虽然 RARP 已被淘汰，但它是理解网络自动配置演进的重要一环，展示了从"只知道 MAC"到"获取 IP 地址"这一基本需求的解决方案。
