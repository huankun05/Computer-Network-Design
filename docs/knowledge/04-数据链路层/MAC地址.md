# MAC 地址（介质访问控制地址）

## 基本信息
| 属性 | 值 |
|------|-----|
| 名称 | MAC 地址（Media Access Control Address） |
| 所属层 | 数据链路层 |
| 长度 | 48 位（6 字节） |
| 表示方式 | 十六进制，6 组用 "-" 或 ":" 分隔 |
| 分配机构 | IEEE |

## 地址格式
```
00-11-22-33-44-55
│           │
│           └── 设备标识（3 字节）
│               由制造商分配
│
└── OUI（组织唯一标识符，3 字节）
     由 IEEE 分配给设备制造商
```

示例：
| OUI | 制造商 |
|-----|--------|
| 00-1A-A0 | Cisco |
| 00-50-56 | VMware |
| 08-00-27 | Intel |
| AC-87-A3 | Apple |

## MAC 地址类型

### 单播地址（Unicast）
- 第一个字节的最低位 = 0
- 示例：`00-11-22-33-44-55`
- 发往特定一台设备

### 组播地址（Multicast）
- 第一个字节的最低位 = 1
- 示例：`01-00-5E-00-00-01`
- 发往特定一组设备

### 广播地址（Broadcast）
- 48 位全为 1
- `ff-ff-ff-ff-ff-ff`
- 发往局域网内所有设备

## 如何查看 MAC 地址
| 操作系统 | 命令 |
|----------|------|
| Windows | `ipconfig /all`（物理地址） |
| Linux | `ip link show` 或 `ifconfig` |
| macOS | `ifconfig` |

## MAC 地址 vs IP 地址
| 特性 | MAC 地址 | IP 地址 |
|------|----------|---------|
| 长度 | 48 位 | 32 位（IPv4） |
| 所在层 | 数据链路层 | 网络层 |
| 作用范围 | 局域网内 | 跨网络 |
| 分配方式 | 硬件固化 + 可修改 | 逻辑分配（手动/DHCP） |
| 是否可变 | 可修改（软件） | 随网络变化 |
| 全局唯一性 | 理论唯一（有例外） | 可重复（私有地址） |

> 类比：MAC 地址像是身份证号（固定），IP 地址像是邮寄地址（随位置变化）

## ARP 与 MAC 地址
- ARP 协议将 IP 地址解析为 MAC 地址
- ARP 缓存表存储 IP ↔ MAC 映射
- 在局域网内通信必须知道 MAC 地址

## MAC 地址在数据帧中的位置
```
| 目的MAC(6B) | 源MAC(6B) | 类型(2B) | 数据 | FCS(4B) |
```
每经过一跳路由器，源 MAC 和目的 MAC 都会改变（重新封装），但 IP 地址不变。
