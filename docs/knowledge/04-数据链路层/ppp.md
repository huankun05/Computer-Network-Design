# PPP 协议（点对点协议）

## 基本信息
| 属性 | 值 |
|------|-----|
| 协议名称 | PPP（Point-to-Point Protocol） |
| 所属层 | 数据链路层 |
| 用途 | 在两个节点之间建立直接的链路层连接 |
| RFC 文档 | RFC 1661 |

## 协议简介
PPP 是用于在两个网络节点之间建立直接连接的数据链路层协议。最典型的应用场景是通过电话线（拨号上网）或光纤（PPPoE 宽带上网）连接 ISP。

## PPP 协议栈
```
    ┌──────────────┐
    │   IP / IPX   │  ← 网络层
    ├──────────────┤
    │    NCP       │  ← 网络控制协议
    ├──────────────┤
    │    LCP       │  ← 链路控制协议
    ├──────────────┤
    │  HDLC 封装    │  ← 数据链路层
    ├──────────────┤
    │  物理介质     │  ← 物理层
    └──────────────┘
```

## PPP 组件

### LCP（链路控制协议）
负责建立、配置、测试和终止数据链路连接：
- 链路建立
- 认证方式协商（PAP / CHAP）
- 链路质量监测
- 链路终止

### NCP（网络控制协议）
为不同的网络层协议提供配置支持：
| NCP 类型 | 支持的协议 |
|----------|-----------|
| IPCP | IPv4 地址分配 |
| IPV6CP | IPv6 配置 |

### 认证协议
| 协议 | 全称 | 特点 |
|------|------|------|
| PAP | 密码认证协议 | 明文传输密码，不安全 |
| CHAP | 挑战握手认证协议 | 三次握手，不传输密码本身 |

## PPP 工作状态
```
Dead ──→ Establish ──→ Authenticate ──→ Network ──→ Open
  ↑         ↑               ↑              ↑          │
  │         │               │              │          │
  └─────────┴───────────────┴──────────────┴──────────┘
                     Terminate
```

| 状态 | 说明 |
|------|------|
| Dead | 链路不可用 |
| Establish | LCP 协商中 |
| Authenticate | 认证中（可选） |
| Network | NCP 配置网络层参数 |
| Open | 链路正常运行 |
| Terminate | 链路关闭中 |

## PPP 帧格式
```
| 标志(1) | 地址(1) | 控制(1) | 协议(2) | 数据(变长) | FCS(2/4) | 标志(1) |
  0x7E      0xFF     0x03              (MTU 1500)
```

## PPPoE（以太网上的 PPP）
- 广泛用于宽带拨号上网
- 将 PPP 帧封装在以太网帧中
- 支持认证、计费、IP 分配
- 中国家庭宽带普遍使用 PPPoE

## 与其他协议的关系
- PPP 与以太网同为数据链路层协议，但场景不同
- PPP 用于点对点连接（如拨号、专线）
- 以太网用于多路访问局域网
