# 交换机（Switch）

## 基本信息
| 属性 | 值 |
|------|-----|
| 设备名称 | 交换机（Switch） |
| 所属层 | 数据链路层（二层交换机） |
| 工作依据 | MAC 地址表 |
| 转发方式 | 存储转发 / 直通转发 |

## 交换机工作原理

### MAC 地址表
交换机维护一张 MAC 地址表，记录 MAC 地址与端口的映射关系：

| MAC 地址 | 端口 | 类型 | 老化时间 |
|----------|------|------|---------|
| 00-11-22-33-44-cc | 端口4 | 动态学习 | 300 秒 |
| 00-11-22-33-44-bb | 端口1 | 动态学习 | 300 秒 |

### 转发三步曲

**1. 学习（Learning）**
交换机收到数据帧时，记录源 MAC 地址和接收端口到 MAC 地址表。

**2. 转发/泛洪（Forwarding / Flooding）**
- 查表命中（已知单播）：从对应端口定向转发
- 查表未命中（未知单播）：从除接收端口外的所有端口泛洪
- 目的 MAC 为广播地址：泛洪

**3. 过滤（Filtering）**
如果目的 MAC 所在端口就是接收端口，则丢弃该帧（无需转发到同一端口）。

### 转发行为示例
```
初始状态: MAC 地址表为空

步骤 1: H1 (端口4) 发送帧给 H2 (端口3)
  → 学习: MAC_cc → 端口4
  → 泛洪: 帧从端口1、2、3 发出
  → H2 收到帧

步骤 2: H2 (端口3) 回复帧给 H1 (端口4)
  → 学习: MAC_dd → 端口3
  → 定向转发: 查表 cc→端口4，仅从端口4 转发

步骤 3: 后续 H1↔H2 通信
  → 两端 MAC 都已在表中
  → 全部定向转发，不再泛洪
```

## 交换机 vs 集线器
| 特性 | 交换机 | 集线器 |
|------|--------|--------|
| 工作层 | 数据链路层 | 物理层 |
| 转发依据 | MAC 地址表 | 无（广播） |
| 冲突域 | 每端口独立 | 所有端口共享 |
| 广播域 | 所有端口同一广播域 | 所有端口同一广播域 |
| 带宽 | 每端口独享 | 所有端口共享 |
| 双工模式 | 全双工 | 半双工 |
| 安全性 | 较高 | 低（所有端口都能收到） |

## 交换机类型
| 类型 | 工作层 | 功能 |
|------|--------|------|
| 二层交换机 | 数据链路层 | 基于 MAC 地址转发 |
| 三层交换机 | 网络层 | 支持路由功能（VLAN 间路由） |
| 多层交换机 | 多层 | 支持更高层的策略转发 |

## 生成树协议（STP）
- 目的：防止二层环路导致广播风暴
- 通过阻塞冗余链路，确保无环拓扑
- 当主链路故障时，自动启用备用链路

## 与其他设备的关系
- 交换机连接同一局域网内的设备（主机、DNS 服务器等）
- 路由器连接不同局域网，隔离广播域
- 在综合场景中，交换机 S 连接 H1、H2、DNS 服务器和路由器 R
