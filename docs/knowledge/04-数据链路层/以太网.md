# 以太网（Ethernet）

## 基本信息
| 属性 | 值 |
|------|-----|
| 标准 | IEEE 802.3 |
| 所属层 | 数据链路层 + 物理层 |
| MAC 地址长度 | 48 位（6 字节） |
| 最小帧长 | 64 字节 |
| 最大帧长 | 1518 字节 |

## MAC 地址
MAC（Media Access Control）地址是网络设备的物理地址，全球唯一。

### 格式
```
00-11-22-33-44-55
|  厂商标识  | 设备标识 |
   3字节       3字节
```

### 特殊地址
| 地址 | 说明 |
|------|------|
| ff-ff-ff-ff-ff-ff | 广播地址 |
| 00-00-00-00-00-00 | 未知地址 |

## 以太网帧格式（Ethernet II）
```
| 目的MAC(6) | 源MAC(6) | 类型(2) | 数据(46-1500) | FCS(4) |
```

| 字段 | 说明 |
|------|------|
| 目的 MAC | 接收方的 MAC 地址 |
| 源 MAC | 发送方的 MAC 地址 |
| 类型 | 上层协议类型（0x0800=IP, 0x0806=ARP） |
| 数据 | 载荷数据（46-1500 字节） |
| FCS | 帧校验序列（CRC 校验） |

## 以太网工作原理

### CSMA/CD（载波监听多路访问/冲突检测）
1. 发送前监听信道是否空闲
2. 空闲时发送数据
3. 发送时检测冲突
4. 检测到冲突后停止发送，发送 48 位干扰信号（Jam Signal）
5. 发送方等待随机时间（二进制指数退避算法）后重试

> **对比现代交换式以太网**：现代网络普遍使用交换机替代集线器，每个端口独占冲突域，支持全双工通信，因此基本不再依赖 CSMA/CD。

## 以太网帧的封装过程
```
应用层数据
    ↓ TCP/UDP 封装
传输层数据段
    ↓ IP 封装
IP 数据报
    ↓ 以太网帧封装（添加 MAC 头 + FCS 尾）
以太网帧
    ↓ 物理层发送
比特流
```

## 与其他协议的关系
- 以太网帧封装 IP 数据报（类型字段 = 0x0800）
- 以太网帧封装 ARP 报文（类型字段 = 0x0806）
- 交换机根据 MAC 地址表转发以太网帧
- 在综合场景中，所有局域网内的帧都通过以太网传输
