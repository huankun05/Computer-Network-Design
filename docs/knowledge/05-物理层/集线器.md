# 集线器（Hub）

## 基本信息
| 属性 | 值 |
|------|-----|
| 设备名称 | 集线器（Hub） |
| 所属层 | 物理层 |
| 工作方式 | 信号再生和广播 |
| 端口数 | 常见 4/8/16/24 口 |

## 工作原理
- 集线器接收到一个端口的信号后，进行放大和整形
- 然后从**除接收端口外的所有端口**广播出去
- 所有端口共享带宽，属于**同一个冲突域**
- 同一时刻只允许一个端口发送数据（半双工）

## 冲突域问题
```
集线器连接的主机都在同一个冲突域内：
  [Hub]
  / | \
H1 H2 H3

H1 发送 → 集线器广播给 H2、H3
H2 同时发送 → 发生冲突！
→ 双方停止，等待随机时间重试（CSMA/CD）
```

## 集线器 vs 交换机
| 特性 | 集线器 | 交换机 |
|------|--------|--------|
| 工作层 | 物理层 | 数据链路层 |
| 转发方式 | 无条件广播 | 按 MAC 地址定向转发 |
| MAC 地址学习 | 无 | 有 |
| 冲突域 | 所有端口共享 | 每端口独立 |
| 双工模式 | 半双工 | 全双工 |
| 带宽分配 | 所有端口共享 | 每端口独享 |
| 安全性 | 低 | 较高 |

## 现状
- 集线器已被交换机**完全取代**
- 现代局域网中几乎不再使用
- 仅在学习网络原理时作为对比参考
