# 综合场景：H1 访问 www.abc.com

## 网络拓扑
```
                        Internet
                           |
                      Web 服务器
                    (www.abc.com)
                           |
                        [R] 路由器
                    192.168.1.1/25
                    00-11-22-33-44-aa
                           |
                        [S] 交换机
                       /   |    \     \
                  端口1  端口2  端口3  端口4
                    |      |      |      |
               DNS服务器   H2     H1
          192.168.1.126/25   |   192.168.1.3/25
          00-11-22-33-44-bb  |   00-11-22-33-44-dd
                            |
                      192.168.1.2/25
                      00-11-22-33-44-cc
```

## 设备信息
| 设备 | IP 地址 | MAC 地址 | 连接端口 |
|------|---------|---------|---------|
| H1 | 192.168.1.2/25 | 00-11-22-33-44-cc | 交换机端口4 |
| H2 | 192.168.1.3/25 | 00-11-22-33-44-dd | 交换机端口3 |
| DNS 服务器 | 192.168.1.126/25 | 00-11-22-33-44-bb | 交换机端口1 |
| 路由器 R | 192.168.1.1/25 | 00-11-22-33-44-aa | 交换机端口2 |
| Web 服务器 | Internet 中 | - | - |

## 初始条件
- H1 的 ARP 缓存表为空
- 交换机 S 的 MAC 地址表为空
- t0 到 t1 期间，网络中无与此次 Web 访问无关的通信

## 通信步骤拆解

### 阶段一：DNS 解析准备（获取 DNS 服务器 MAC 地址）

**步骤 1：H1 检查本地 DNS 缓存**
- 起点：H1
- 终点：H1 本地
- 协议：无（本地操作）
- 广播/单播：无
- 说明：H1 需要解析 www.abc.com，检查本地 DNS 缓存，未命中

**步骤 2：H1 发送 ARP 请求（查询 DNS 服务器 MAC）**
- 起点：H1
- 终点：广播
- 协议：ARP
- 广播/单播：广播
- 源 MAC：00-11-22-33-44-cc
- 目的 MAC：ff-ff-ff-ff-ff-ff
- 源 IP：192.168.1.2
- 目的 IP：192.168.1.126
- 交换机行为：学习 MAC cc → 端口4，从端口1、2、3 泛洪
- 交换表变化：新增 cc → 端口4

**步骤 3：DNS 服务器回复 ARP 响应**
- 起点：DNS 服务器
- 终点：H1
- 协议：ARP
- 广播/单播：单播
- 源 MAC：00-11-22-33-44-bb
- 目的 MAC：00-11-22-33-44-cc
- 源 IP：192.168.1.126
- 目的 IP：192.168.1.2
- 交换机行为：学习 MAC bb → 端口1，查表转发到端口4
- 交换表变化：新增 bb → 端口1
- H1 ARP 表变化：新增 192.168.1.126 → 00-11-22-33-44-bb

### 阶段二：DNS 查询与响应

**步骤 4：H1 发送 DNS 查询**
- 起点：H1
- 终点：DNS 服务器
- 协议：DNS（UDP 53）
- 广播/单播：单播
- 源 MAC：cc
- 目的 MAC：bb
- 源 IP：192.168.1.2
- 目的 IP：192.168.1.126
- 交换机行为：查表转发到端口1

**步骤 5：DNS 服务器处理查询并回复**
- 起点：DNS 服务器
- 终点：H1
- 协议：DNS（UDP 53）
- 广播/单播：单播
- 源 MAC：bb
- 目的 MAC：cc
- 源 IP：192.168.1.126
- 目的 IP：192.168.1.2
- 说明：DNS 服务器解析 www.abc.com，返回对应 IP 地址
- H1 DNS 缓存变化：新增 www.abc.com → 解析结果 IP

### 阶段三：获取网关 MAC 地址

**步骤 6：H1 发送 ARP 请求（查询网关 MAC）**
- 起点：H1
- 终点：广播
- 协议：ARP
- 广播/单播：广播
- 源 MAC：cc
- 目的 MAC：ff-ff-ff-ff-ff-ff
- 源 IP：192.168.1.2
- 目的 IP：192.168.1.1
- 交换机行为：广播泛洪

**步骤 7：路由器 R 回复 ARP 响应**
- 起点：路由器 R
- 终点：H1
- 协议：ARP
- 广播/单播：单播
- 源 MAC：aa
- 目的 MAC：cc
- 源 IP：192.168.1.1
- 目的 IP：192.168.1.2
- H1 ARP 表变化：新增 192.168.1.1 → 00-11-22-33-44-aa

### 阶段四：TCP 连接建立

**步骤 8：H1 发送 TCP SYN（三次握手第一步）**
- 起点：H1
- 终点：Web 服务器（经路由器）
- 协议：TCP
- 广播/单播：单播
- 源 MAC：cc → 目的 MAC：aa（H1 → 路由器）
- 源 IP：192.168.1.2 → 目的 IP：Web 服务器 IP
- 说明：SYN=1, seq=x

**步骤 9：Web 服务器回复 SYN+ACK（三次握手第二步）**
- 起点：Web 服务器
- 终点：H1（经路由器）
- 协议：TCP
- 说明：SYN=1, ACK=1, seq=y, ack=x+1

**步骤 10：H1 发送 ACK（三次握手第三步）**
- 起点：H1
- 终点：Web 服务器
- 协议：TCP
- 说明：ACK=1, seq=x+1, ack=y+1，连接建立

### 阶段五：HTTP 请求与响应

**步骤 11：H1 发送 HTTP GET 请求**
- 协议：HTTP
- 说明：通过已建立的 TCP 连接发送 HTTP GET 请求

**步骤 12：Web 服务器返回 HTTP 响应**
- 协议：HTTP
- 说明：Web 服务器处理请求，返回 HTTP 响应

### 阶段六：TCP 连接释放

**步骤 13-16：TCP 四次挥手**
- 协议：TCP
- 说明：FIN → ACK → FIN → ACK，连接释放

## 最终状态
- H1 的 ARP 表：包含 DNS 服务器和路由器的 IP-MAC 映射
- 交换机 S 的 MAC 地址表：包含 H1、DNS 服务器、路由器的 MAC-端口映射
- H1 已获得 www.abc.com 的 IP 地址
- HTTP 通信完成
